سیستم مدیریت آپلود فایل با Factory Method
---------------------------------------
این پروژه فقط یک اپلیکیشن به نام main دارد.
----------
دو مدل برای دو کلاس جهت آپلود فایل و آپلود متافایل و دو سریالایزر برای این مدل ها تعریف شده است. در کلاس مربوط به متادیتا، از یک ولیدیتور برای کنترل حجم استفاده شده است.
---------
در فایل views این اپلیکیشن دو کلاس نوشته شده است. 
یک کلاس برای آپلود فایل، بازیابی فایل ، حذف فایل و درج فایل. 

یک کلاس دیگر برای آپلود متادیتا نوشته شده است. در این کلاس از ویو-ست های برای آپلود انواع فایل استفاده شده است. 
که url های مربوط به هر کدام در فایل urls.py آمده است.

یک کلاس فکتوری در فایل factory.py ایجاد شده است که برای فایل های تصویر و داکیومنت تعریف شده و قابل توسعه است.
از این کلاس فکتوری در کلاس متادیتا استفاده شده است.

-------------
هر دو این کلاس ها از permission ها استفاده می کنند.
یک سوپریوز به نام admin و پسورد admin تعریف شده است که با فراخوانی api-token-auth/ می توان برای این یوزر توکن ایجاد کرد و با استفاده از این توکن می توان در پستمن به کلاس های ویو با متودهای گت و پست و ... دسترسی پیدا کرد.

-------------
ی کلاس فکتوری برای انواع ذخیره سازی ایجاد شد که با توجه به مشکلات استوریج گوگل و آمازون و تحریم ایران قادر به تست و استفاده از آنها آنها نبودم.
